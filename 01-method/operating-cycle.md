# The Operating Cycle (Ω)

Ω is the continuous application of four components with feedback loops:

```
   +--> LEARN --> ALIGN --> PERFORM --> REVIEW --+
   |                                             |
   +------------------ feedback <----------------+

   REVIEW feeds LEARN (what changed) and ALIGN (what to redesign).
   PERFORM emits events to REVIEW continuously.
   LEARN fires off-cycle whenever context triggers trip.
```

The single most common misreading of the model is to treat Ω as a project: run LEARN once, ALIGN once, stand up PERFORM, schedule an audit and declare completion. Ω is an operating rhythm. Some elements run continuously, some periodically and some only on trigger. A capability is alive when all three modes are observable.

## What each component produces

| Component | Question it answers | Core artifacts |
|---|---|---|
| LEARN | What is true about our world, our organization, our culture and our stakeholders? | External and internal context briefs, culture assessment, stakeholder map, watchlist with triggers |
| ALIGN | Given that, what are we trying to achieve, what stands in the way or helps and how will we address it? | Direction and appetite statements, objectives register, identified and assessed registers, capability and treatment design |
| PERFORM | Are the designed actions actually operating? | Operating controls, policy set, communication and training records, incentive design, intake log, investigation files, response records |
| REVIEW | Is it working, can we prove it and what must change? | Indicator dashboards, assurance plan and reports, improvement backlog with closure evidence |

## Default cadence

Calibrate to your size and risk profile; the defaults below are a starting position, not a rule.

| Elements | Default cadence | Trigger-based additions |
|---|---|---|
| L1 External Context | Quarterly scan | New law or enforcement action, market shock, technology shift |
| L2 Internal Context | Annual refresh | Reorganization, acquisition, new system or product |
| L3 Culture | Annual assessment plus pulse checks | Spike in turnover, incidents or speak-up anomalies |
| L4 Stakeholders | Annual refresh | New market, major deal, regulatory engagement |
| A1 Direction | Annual, with the board | Strategy change, leadership change |
| A2 Objectives | Annual with strategy, quarterly review | Material context change from LEARN |
| A3 Identification / A4 Assessment | Semiannual full pass, rolling additions | Any new objective, obligation or significant event |
| A5 Design | On every treatment decision | Findings from REVIEW |
| P1 Controls | Continuous, per-control frequency | Control failure, process change |
| P2 Policies | Lifecycle-driven; review each policy at least every two years | Obligation change, incident lesson |
| P3 Communication / P4 Education | Ongoing plan; role-based training annually and at onboarding | New policy, new risk, new role |
| P5 Incentives | Annual, aligned to the compensation cycle | Evidence of perverse incentives |
| P6 Notification | Continuous intake | Never gated; always open |
| P7 Inquiry | On notification or detection | Proactive inquiry on risk signals |
| P8 Response | On confirmed issue | Crisis activation criteria |
| R1 Monitoring | Continuous, reported monthly | Threshold breach escalates immediately |
| R2 Assurance | Per risk-based assurance plan | Post-incident, pre-transaction, regulator request |
| R3 Improvement | Monthly backlog triage, quarterly change review | Any accepted finding |

## Accountability

The cycle needs named owners, not a shared inbox. A generic allocation, to be adapted:

| Work | Board | Executive owner | GRC function | Unit management | Independent assurance |
|---|---|---|---|---|---|
| Set direction and appetite (A1) | A | R | C | C | I |
| Own objectives (A2) | I | A | C | R | I |
| Identify and assess (A3, A4) | I | A | R (facilitates) | R (contributes) | I |
| Design treatments (A5) | I | A | R | C | C |
| Operate controls and policies (P1, P2) | I | A | C (designs, monitors) | R (operates) | I |
| Communicate, train, incent (P3 to P5) | I | A | R | C | I |
| Intake, investigate, respond (P6 to P8) | I | A | R | C | I |
| Monitor (R1) | I | A | R | C | I |
| Independent assurance (R2) | A | I | C | C | R |
| Drive improvement (R3) | I | A | R | R | C |

R responsible, A accountable, C consulted, I informed. The pattern to preserve: management operates, the GRC function designs and monitors, independent assurance (typically internal audit) examines both without owning either. Collapsing the third role into the second quietly converts assurance into self-grading.

## The minimum viable cycle

For a small organization or a single-person function, the cycle does not shrink by deleting components; it shrinks by reducing depth and formality while keeping all four alive:

1. LEARN: one context brief, refreshed quarterly in an afternoon, with a five-item watchlist.
2. ALIGN: one page of objectives with owners; one combined register of the top risks, opportunities and obligations, assessed on a simple calibrated scale.
3. PERFORM: the ten controls that matter, each with an owner and evidence; a short policy set; an intake channel that is genuinely answered.
4. REVIEW: a one-page monthly dashboard; one independent look per year, even if outsourced; a visible improvement list.

What is not optional at any size: objectives linkage, an open intake channel and closure evidence for improvements.

## Failure modes of the cycle

1. **Control theater.** PERFORM without ALIGN. Controls exist because a checklist said so, not because an assessed item required them. Symptom: nobody can name the risk or obligation behind a control.
2. **Strategy in a vacuum.** ALIGN without LEARN. Objectives and assessments built on last year's world. Symptom: the register never changes and never predicts anything.
3. **Analysis without alignment.** LEARN without ALIGN. Excellent scanning that never alters an objective, an assessment or a design. Symptom: beautiful briefs, unchanged registers.
4. **Audit fatigue loop.** REVIEW without R3. Findings are issued, accepted and reissued. Symptom: repeat findings with new dates.
5. **The frozen register.** A3 and A4 run once, then the register becomes furniture. Symptom: assessment dates older than the last reorganization.
6. **The unowned cycle.** Everything above at once, because no named person is accountable for Ω itself. Symptom: the question "who owns the cycle" produces a meeting instead of a name.

## Reading the equation against the cycle

Ω is the engine; C is what the engine is built from; SP is where the key first turns; UO is the direction the vehicle must actually travel. A cycle can spin while the vehicle stands still. The Universal Outcomes, evidenced in `principled-performance.md`, are how you check for motion.
